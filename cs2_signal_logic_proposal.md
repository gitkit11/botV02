## Предложение по обновлению логики сигналов CS2 в `signal_engine.py`

Как мы обсуждали, текущая логика сигналов CS2 в `signal_engine.py` не учитывает многие «скрытые» факторы, которые критичны для киберспорта. Чтобы сделать сигналы более качественными и точными, я предлагаю значительно расширить критерии оценки матчей CS2.

### 1. Текущая логика `check_cs2_signal`

Сейчас функция `check_cs2_signal` оценивает матч по 7 критериям, каждый из которых дает 1 балл:
1.  Вероятность (prob >= 55%)
2.  EV (ev >= 7%)
3.  Коэффициент (1.45 <= odds <= 3.50)
4.  Форма (wins >= 3 из 5)
5.  Разница ELO (elo_fav - elo_opp >= 30)
6.  MIS (Map Influence Score) - разница по картам (mis_fav - mis_opp >= 3%)
7.  Рейтинг игроков (rat_fav - rat_opp >= 0.06)

Минимальный проходной балл — 4 из 7.

### 2. Предлагаемые новые параметры для `check_cs2_signal`

Для учета «скрытых» факторов, функция `check_cs2_signal` должна будет принимать больше данных. Эти данные будут предварительно собираться в `main.py` (или в новом модуле-оркестраторе) с использованием `hltv_stats.py`, `veto_logic.py` и `pandascore.py`.

**Новые параметры (в дополнение к существующим):**
*   `home_map_winrates: dict` - Словарь винрейтов домашней команды по картам (например, `{'Dust2': 0.7, 'Inferno': 0.6}`).
*   `away_map_winrates: dict` - Аналогично для гостевой команды.
*   `predicted_maps: list[str]` - Список карт, которые, скорее всего, будут сыграны в матче (после симуляции вето).
*   `home_key_players_form: list[float]` - Список рейтингов ключевых игроков домашней команды (например, 3 лучших игрока).
*   `away_key_players_form: list[float]` - Аналогично для гостевой команды.
*   `ai_cs2_agrees: Optional[bool]` - Флаг, указывающий, подтвердил ли быстрый AI-агент (например, Llama) преимущество одной из команд, основываясь на комплексном анализе.

### 3. Новые критерии оценки и баллы (всего 10 баллов)

Я предлагаю увеличить общее количество критериев до 10, чтобы более точно оценивать матчи. Минимальный проходной балл будет увеличен до **6 из 10**.

**Существующие критерии (7 баллов):**
1.  **Вероятность** (prob >= 55%) - 1 балл
2.  **EV** (ev >= 7%) - 1 балл
3.  **Коэффициент** (1.45 <= odds <= 3.50) - 1 балл
4.  **Форма** (wins >= 3 из 5) - 1 балл
5.  **Разница ELO** (elo_fav - elo_opp >= 30) - 1 балл
6.  **MIS (Map Influence Score)** - разница по картам (mis_fav - mis_opp >= 3%) - 1 балл
7.  **Рейтинг игроков** (rat_fav - rat_opp >= 0.06) - 1 балл

**Новые критерии (3 балла):**

8.  **Преимущество на ключевых картах (Map Pool Advantage)** - **1 балл**
    *   **Логика:** Если после симуляции вето (`veto_logic.simulate_bo3_veto`) определены 2-3 карты, и команда-фаворит имеет на этих картах **значительно более высокий винрейт** (например, средний винрейт на этих картах на 10% выше, чем у оппонента).
    *   **Порог:** `map_advantage_score` (рассчитывается заранее) >= `CS2_CFG["min_map_advantage"]` (например, 0.10).

9.  **Форма ключевых игроков (Key Player Form)** - **1 балл**
    *   **Логика:** Если 2 из 3 ключевых игроков команды-фаворита имеют высокий индивидуальный рейтинг (например, > 1.20) и/или находятся в хорошей форме по последним матчам.
    *   **Порог:** `key_player_advantage` (рассчитывается заранее) >= `CS2_CFG["min_key_player_advantage"]` (например, 0.15).

10. **Подтверждение от быстрого ИИ-агента (AI CS2 Agrees)** - **1 балл**
    *   **Логика:** Если быстрый AI-агент (например, Llama), получив сводку по матчу (команды, коэффициенты, маппул, ключевые игроки), подтверждает наличие «скрытого» преимущества у команды-фаворита.
    *   **Порог:** `ai_cs2_agrees` == `True`.

### 4. Обновленный `CS2_CFG`

Для поддержки новых критериев, словарь `CS2_CFG` будет расширен:

```python
CS2_CFG = {
    "min_prob":             0.55,
    "min_ev":               0.07,
    "min_odds":             1.45,
    "max_odds":             3.50,
    "min_form_wins":        3,
    "min_elo_gap":          30,
    "min_mis_gap":          0.03,
    "min_rating_gap":       0.06,
    "min_map_advantage":     0.10,  # Новый: мин. преимущество на картах (в %) 
    "min_key_player_advantage": 0.15, # Новый: мин. преимущество ключевых игроков (в %) 
    "min_score":            6,     # Изменено: Минимум 6 баллов из 10 для сигнала
}
```

### 5. Как это решает проблему «скрытых» сигналов

Внедрение этих критериев позволит системе:
*   **Учитывать контекст карт:** Бот будет «понимать», что команда сильна не просто так, а на конкретных картах, которые, скорее всего, будут сыграны.
*   **Оценивать индивидуальное мастерство:** Фокус на ключевых игроках поможет выявлять матчи, где индивидуальный скилл может перевесить общую статистику команды.
*   **Использовать «интуицию» ИИ:** Быстрый AI-агент будет выступать в роли «эксперта», который, основываясь на большом объеме данных, может подтвердить или опровергнуть потенциальный сигнал, добавляя человеческий фактор в чисто математическую модель.
