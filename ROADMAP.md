## 🔑 Правила для разработчиков

### Как добавить новый вид спорта:
1. Создай папку `sports/новый_спорт/`
2. Создай `__init__.py` с публичным интерфейсом
3. Добавь кнопку в `main.py` в функцию `build_sport_keyboard()`
4. Добавь обработчик в `handle_text()` и `handle_callback()`
5. **Не трогай** логику футбола в `main.py`

### Зоны ответственности:
| Разработчик | Файлы |
|-------------|-------|
| Главный (этот чат) | main.py, agents.py, math_model.py, database.py |
| CS2 разработчик | sports/cs2/ (только эта папка) |
| Теннис разработчик | sports/tennis/ (только эта папка) |

### Запрещено:
- ❌ Менять `main.py` без согласования с главным разработчиком
- ❌ Хардкодить API ключи в коде
- ❌ Делать `git push --force`

## 📊 Анализ по видам спорта

### Футбол — 5 уровней анализа:
```
1. Пуассон + Dixon-Coles (xG данные) → голевые вероятности
2. ELO + Форма (реальные данные сезона) → сила команд
3. Нейросеть Пророк (66к матчей) → исторические паттерны
4. GPT-4.1 + Llama 3.3 (новости, травмы) → контекстный анализ
5. Ансамбль → итоговые вероятности + value bets
```

### CS2 — 3 уровня анализа:
```
1. Мап-вето симуляция (BO3) → вероятности по картам
2. MIS оценка (Map Impact Score) → итоговые вероятности
3. GPT + Llama → тактический анализ
```

### Теннис (план):
```
1. Статистика подач/приёмов → вероятности геймов
2. Покрытие + рейтинг ATP/WTA → сила игроков
3. AI агент → контекстный анализ
```
