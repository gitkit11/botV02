## План по улучшению сигналов CS2 и внедрению ИИ-самообучения

После детального аудита модулей CS2 (`sports/cs2/`) и базы данных (`database.py`), а также анализа текущей логики формирования сигналов (`signal_engine.py`), я готов предложить архитектуру для внедрения «ИИ-мозга» и значительного улучшения качества сигналов CS2.

### 1. Улучшение качества сигналов CS2: Внедрение «скрытых» факторов

Текущая система сигналов CS2 в `signal_engine.py` использует общие метрики (вероятность, EV, коэффициент, форма, ELO, разница MIS, рейтинг игроков). Чтобы сделать сигналы более «качественными» и учитывать «скрытые» факторы, мы должны интегрировать более глубокий анализ из `hltv_stats.py` и `veto_logic.py`.

**Предлагаемые изменения в `signal_engine.py` (функция `check_cs2_signal`):**

1.  **Учет маппула и вето:**
    *   **Текущая проблема:** `check_cs2_signal` не учитывает сильные/слабые карты команд и процесс вето.
    *   **Решение:** Перед вызовом `check_cs2_signal`, в `main.py` (или в отдельной функции-обертке) необходимо:
        *   Получить статистику по картам для обеих команд из `hltv_stats.MAP_STATS`.
        *   Симулировать процесс вето с помощью `veto_logic.simulate_bo3_veto`, чтобы определить наиболее вероятные карты для игры.
        *   Рассчитать **средний винрейт** команды на этих **предполагаемых картах**.
    *   **Новый критерий для сигнала:** Добавить новый балл за «Преимущество на картах» (например, если разница в среднем винрейте на предполагаемых картах > 10%).

2.  **Учет ключевых игроков:**
    *   **Текущая проблема:** Учитывается только средний рейтинг игроков, но не их роли и индивидуальная форма.
    *   **Решение:** Использовать `hltv_stats.get_key_players` для получения статистики по 2-3 лучшим игрокам каждой команды (AWPer, Star Rifler, IGL).
    *   **Новый критерий для сигнала:** Добавить балл, если ключевой игрок команды находится в хорошей форме (например, рейтинг > 1.25) и его оппонент по роли слабее.

3.  **Интеграция AI-агентов для «быстрого» анализа:**
    *   **Текущая проблема:** AI-агенты (GPT, Llama) используются для глубокого анализа по запросу, но не для массового отбора сигналов.
    *   **Решение:** Создать упрощенный промпт для одного из быстрых AI-агентов (например, Llama через Groq API), который будет получать:
        *   Базовые данные матча (команды, коэффициенты).
        *   Сводку по маппулу (кто на каких картах сильнее).
        *   Сводку по ключевым игрокам.
    *   **Новый критерий для сигнала:** Добавить балл, если AI-агент подтверждает «скрытое» преимущество одной из команд (например, «ИИ видит преимущество Team Spirit на Nuke, основываясь на их винрейте 72.7% и форме donk»). Это будет аналог `ai_agrees` в футболе, но с более глубоким контекстом CS2.

### 2. Внедрение ИИ-самообучения и статистики

Для создания самообучающейся системы, которая будет улучшать себя на основе результатов, нам потребуется модифицировать базу данных и создать новый слой логики.

**Предлагаемые изменения в `database.py`:**

1.  **Разделение таблиц по видам спорта:**
    *   **Текущая проблема:** Единая таблица `predictions` для всех видов спорта, что усложняет специфический анализ.
    *   **Решение:** Создать две отдельные таблицы: `football_predictions` и `cs2_predictions`. Каждая таблица будет иметь свои специфические поля для хранения данных, релевантных только этому виду спорта (например, `map_pool_advantage`, `key_player_rating` для CS2).
    *   **Миграция:** Обеспечить плавный переход для существующих данных.

2.  **Добавление полей для обучения ИИ:**
    *   **Для CS2:** Добавить поля для хранения:
        *   `predicted_maps`: Список карт, предсказанных вето.
        *   `map_advantage_score`: Балл за преимущество на картах.
        *   `key_player_advantage`: Балл за преимущество ключевых игроков.
        *   `ai_signal_reason`: Краткое обоснование от AI-агента, если он дал сигнал.
    *   **Для всех:** Добавить поле `model_weights_at_prediction`: JSON-строка с весами различных моделей (ELO, Prophet, AI-агентов) на момент прогноза. Это критично для понимания, какие модели работали хорошо, а какие — нет.

**Архитектура «ИИ-мозга» (Meta-аналитик):**

Мы создадим новый модуль, например, `meta_learner.py`, который будет работать в фоновом режиме или по расписанию.

1.  **Анализ результатов:** `meta_learner.py` будет периодически (например, раз в день) запрашивать из `database.py` все завершенные матчи, для которых есть прогнозы и реальные результаты.
2.  **Оценка эффективности:** Для каждого прогноза он будет оценивать:
    *   **Точность:** Насколько прогноз совпал с реальным исходом.
    *   **ROI:** Какую прибыль/убыток принесла ставка.
    *   **Вклад моделей:** Какие модели (ELO, Prophet, GPT, Llama, Mixtral) и какие критерии (EV, форма, маппул) были наиболее точными для данного матча.
3.  **Корректировка весов/логики:**
    *   **Для `signal_engine.py`:** ИИ будет анализировать, какие сигналы были успешными, а какие — нет. Например, если сигналы CS2 с высоким баллом за «Преимущество на картах» часто проигрывают, ИИ может предложить снизить вес этого критерия или скорректировать порог.
    *   **Для AI-агентов:** ИИ может генерировать новые, более эффективные промпты для GPT/Llama, основываясь на анализе прошлых успешных и неудачных прогнозов. Например, добавить в промпт: «Учитывай, что команда X часто проигрывает на карте Y, даже если у нее высокий ELO».
    *   **Для математических моделей:** ИИ может предлагать корректировки K-фактора для ELO или параметров Пуассона, если они систематически ошибаются.
4.  **Обновление конфигурации:** ИИ будет записывать свои рекомендации или изменения в специальный файл конфигурации (например, `learning_config.json`), который будет загружаться при старте бота и влиять на работу `signal_engine.py` и других модулей.

### 3. Интеграция в кнопку «Статистика»

Кнопка «Статистика» (`/stats` команда) будет расширена для отображения результатов работы «ИИ-мозга»:

*   **ROI по видам спорта:** Отдельный ROI для футбола и CS2.
*   **Эффективность критериев:** Какие критерии (EV, форма, маппул) наиболее часто приводят к успешным сигналам.
*   **Динамика обучения:** Графики, показывающие, как менялась точность и ROI с течением времени благодаря самообучению.
