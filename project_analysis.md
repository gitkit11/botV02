# Анализ проекта Chimera AI

## 1. Обзор проекта

**Chimera AI** — это Telegram-бот, предназначенный для прогнозирования результатов футбольных матчей с использованием комбинации математических моделей, искусственного интеллекта и анализа данных. Бот стремится находить ставки с положительным ожидаемым значением (EV) для пользователей.

### Ключевые особенности:
*   **Ансамблевый подход**: Использует несколько моделей и агентов для формирования прогноза.
*   **Математические модели**: Включает Пуассона (с поправкой Диксона-Коулза) и ELO-рейтинги.
*   **AI Агенты**: Задействует GPT-4o (Маэстро), Llama 3.3 70B и Gemma2-9b-it для контекстного и тактического анализа.
*   **Анализ новостей и травм**: Использует GNews API и GPT-4o-mini для оценки настроения команд и выявления травм.
*   **Value Bets**: Идентифицирует ставки с EV > 10%, коэффициентом >= 1.55 и уверенностью >= 52%.
*   **ROI трекер**: Отслеживает историю прогнозов, точность моделей и ROI.
*   **CS2 интеграция**: Поддерживает анализ матчей по CS2 (хотя основной фокус на футболе).

## 2. Архитектура и основные компоненты

Проект построен модульно, что позволяет легко добавлять новые модели и агенты. Основные файлы и их функции:

### `main.py`

Это основной файл, который управляет логикой Telegram-бота. Он инициализирует `aiogram` диспетчер, обрабатывает команды пользователя (`/start`, `/results`), а также запросы на анализ матчей и выбор лиг/рынков. В `main.py` происходит координация работы всех агентов и моделей, загрузка данных (ELO, форма команд, модель Prophet), кэширование матчей и форматирование итоговых отчётов.

**Ключевые функции:**
*   Инициализация `aiogram` бота и диспетчера.
*   Обработка команд `/start`, `/results`.
*   Загрузка и обновление ELO-рейтингов и формы команд.
*   Загрузка модели `Prophet` (TensorFlow/Keras).
*   Получение списка матчей через The Odds API.
*   Формирование интерактивных клавиатур для Telegram.
*   Координация вызовов AI-агентов и математических моделей.
*   Форматирование и отправка отчётов пользователю.

### `agents.py`

Содержит логику взаимодействия с различными AI-моделями (GPT, Llama, Gemma2) и их специализацию. Здесь определены промпты для каждого агента и функции для их вызова. Также реализована логика ансамблирования прогнозов от разных моделей.

**Ключевые агенты:**
*   **Statistician Agent (gpt-4.1-mini)**: Анализирует числовые данные и статистику.
*   **Scout Agent (gpt-4.1-mini)**: Анализирует новости и настроения команд.
*   **Arbitrator Agent (gpt-4.1-mini)**: Объединяет отчёты Статистика и Разведчика, находит Value Bets и рассчитывает критерий Келли.
*   **Llama Agent (llama-3.3-70b-versatile)**: Предоставляет независимое мнение, прогнозы по тоталам и "обе забьют". Использует Groq API.
*   **Mixtral Agent (llama-3.1-8b-instant)**: Третий независимый агент для тактического анализа и паттернов. Также использует Groq API.
*   **Market Agents**: Специализированные агенты для анализа рынков голов, угловых, карточек и гандикапов.

### `math_model.py`

Реализует математические модели прогнозирования.

**Ключевые модели:**
*   **ELO Rating**: Динамическая система рейтингов команд, учитывающая домашнее преимущество и форму последних 5 матчей. Рейтинги сохраняются в `elo_ratings.json`.
*   **Poisson Distribution (Dixon-Coles)**: Расчёт вероятностей точного счёта, исходов и тоталов на основе ожидаемых голов (xG) с поправкой Диксона-Коулза для повышения точности прогнозов малых счётов.
*   **Expected Goals (xG)**: Расчёт ожидаемых голов для каждой команды в матче на основе их атакующих и защитных xG-показателей, с регрессией к среднему и домашним преимуществом.

### `database.py`

Отвечает за взаимодействие с базой данных SQLite (`chimera_predictions.db`). Хранит историю прогнозов, результаты матчей, статистику по моделям и ROI.

**Ключевые функции:**
*   `init_db()`: Инициализация базы данных и создание таблицы `predictions`.
*   `save_prediction()`: Сохранение детального прогноза матча.
*   `update_result()`: Обновление результатов матчей и расчёт точности прогнозов и ROI.
*   `get_statistics()`: Получение общей статистики по прогнозам.

### `config.py`

Управляет загрузкой API-ключей и других конфигурационных параметров из `.env` файла, обеспечивая безопасность и гибкость настроек.

## 3. Зависимости

Проект использует следующие основные библиотеки (согласно `requirements.txt`):
*   `aiogram`: Для создания Telegram-бота.
*   `tensorflow`: Для загрузки и использования модели `Prophet`.
*   `numpy`, `pandas`, `scikit-learn`: Для работы с данными и предобработки.
*   `openai`: Для взаимодействия с OpenAI API (GPT-4o).
*   `python-dotenv`: Для загрузки переменных окружения из `.env` файла.
*   `requests`: Для HTTP-запросов к внешним API (The Odds API, GNews API).

## 4. Общие впечатления и потенциальные улучшения

Проект **Chimera AI** представляет собой сложную и хорошо структурированную систему для спортивного прогнозирования. Использование ансамбля из математических моделей и нескольких AI-агентов является сильной стороной, позволяющей учитывать различные аспекты матча (статистика, новости, тактика).

### Потенциальные области для улучшения:
*   **Обработка ошибок и логирование**: Хотя логирование присутствует, можно улучшить обработку специфических ошибок API и добавить более детальные логи для отладки сложных взаимодействий между агентами.
*   **Масштабируемость**: При увеличении количества пользователей и матчей, стоит рассмотреть более производительные решения для базы данных (например, PostgreSQL) и асинхронной обработки запросов к внешним API.
*   **Конфигурация агентов**: Возможность динамической настройки весов агентов или их промптов без изменения кода может быть полезной.
*   **CS2 интеграция**: Расширить функционал CS2, так как сейчас он выглядит менее проработанным по сравнению с футбольным анализом.
*   **UI/UX**: Для Telegram-бота всегда есть возможности улучшить пользовательский интерфейс, например, добавить больше интерактивных элементов или более наглядные графики статистики.
