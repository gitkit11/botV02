# Анализ архитектуры Chimera AI и План доработки до "Идеала"

## 1. Текущее состояние (Аудит)

Проект представляет собой высокоуровневую систему прогнозирования спортивных событий (Футбол, CS2) с использованием ансамбля методов.

### Сильные стороны:
- **Многослойный анализ:** Сочетание математических моделей (Пуассон, ELO), нейросетей (LSTM "Пророк") и LLM-агентов (GPT-4o, Llama 3.3).
- **Контекстная осведомленность:** Интеграция новостей (GNews), травм и настроений команд (Sentiment Analysis).
- **Специфика видов спорта:** Для CS2 реализована симуляция мап-вето и учет индивидуального рейтинга игроков (HLTV).
- **Система сигналов:** Балльная система (Signal Engine) позволяет гибко настраивать пороги входа в сделку.

### Выявленные проблемы и узкие места:
1.  **Статичность ИИ-мозга:** Файл `meta_learner.py` содержит зачатки логики, но не интегрирован в основной цикл обучения. Он лишь "предлагает" изменения, но не закрывает цикл обратной связи автоматически.
2.  **Зависимость от внешних API:** Высокая зависимость от The Odds API, PandaScore и GNews. При сбоях или исчерпании лимитов бот теряет точность.
3.  **Отсутствие глубокого обучения на результатах:** Модель "Пророк" статична. Она не дообучается на новых матчах, которые проходят ежедневно.
4.  **CS2 Маппул:** Использование Mock-данных (`TEAM_MAP_PREFERENCES`) для предпочтений команд снижает точность симуляции вето для команд вне топ-10.
5.  **Теннис:** Модуль находится в состоянии заглушки.

## 2. Целевая архитектура ("Идеал")

Идеальное состояние проекта — это **полностью автономная самообучающаяся система**, которая:
1.  **Закрывает цикл:** Автоматически проверяет результаты -> Анализирует ошибки -> Корректирует веса ансамбля.
2.  **Минимизирует Mock-данные:** Заменяет статические предпочтения динамическим парсингом или накопленной базой.
3.  **Масштабируется:** Легко добавляет новые рынки (угловые, карточки, азиатские гандикапы).

## 3. Пошаговый план реализации

### Фаза 1: Интеграция Meta-Learner (Самообучение)
- [ ] **Автоматизация весов:** Доработать `agents.py`, чтобы веса моделей в `build_math_ensemble` брались из БД или конфига, обновляемого `MetaLearner`.
- [ ] **Логирование факторов:** Сохранять в БД не только прогноз, но и все входные параметры (ELO gap, xG diff, Sentiment score), чтобы Meta-Learner мог понять, какой фактор "врет".
- [ ] **Cron-задача:** Настроить ежедневный запуск `meta_learner.py` для анализа вчерашних матчей.

### Фаза 2: Улучшение CS2 модуля
- [ ] **Динамический маппул:** Реализовать сохранение статистики карт из PandaScore/HLTV в локальную БД, чтобы не зависеть от Mock-словарей.
- [ ] **Учет замен:** Интегрировать проверку составов непосредственно перед матчем (через HLTV API или парсинг), так как замены в CS2 критично меняют шансы.

### Фаза 3: Расширение рынков и видов спорта
- [ ] **Теннис:** Реализовать базовый расчет на основе рейтинга ATP/WTA и статистики покрытий.
- [ ] **Новые рынки:** Добавить в футбол Азиатский гандикап и Тоталы угловых (база для этого уже есть в `agents.py`).

### Фаза 4: Отказоустойчивость и мониторинг
- [ ] **Fallback системы:** Добавить альтернативные источники данных (например, RapidAPI Football) на случай падения основного API.
- [ ] **Dashboard:** Создать админ-команду в боте для визуализации текущего ROI и весов моделей в реальном времени.
